# Contributing Guide

Thank you for considering a contribution.

## Getting Started

1. Fork the repository.
2. Create a feature branch from `main`:
   ```bash
   git checkout -b feat/your-change
   ```
3. Make your changes in small, reviewable commits.

## Development Workflow

- Keep changes focused on one concern per pull request.
- Write clear commit messages using the imperative mood (for example: `Add input validation for user profile form`).
- Update documentation when behavior or interfaces change.

## Code Quality

Before opening a pull request, run any available checks and tests in your environment.

Examples:

```bash
# Replace with project-specific commands
npm test
npm run lint
```

If a command does not apply to this repository yet, include a short note in your PR describing what you validated manually.

## Pull Request Checklist

Please include:

- A concise summary of the change.
- Why the change is needed.
- Any testing notes (commands run, manual verification, and outcomes).
- Screenshots or recordings for UI changes (if applicable).

## Reporting Issues

When filing a bug report, include:

- Expected behavior
- Actual behavior
- Steps to reproduce
- Relevant logs or error messages
- Environment details (OS, runtime, versions)

## Code of Conduct

Be respectful and constructive in all project interactions.
